# 特殊成员卡「夹」成员卡功能设计文档

> 文档类型：设计文档
> 适用范围：特殊成员卡下方堆叠成员卡的规则边界、数据模型、联机投影和 UI 展示
> 当前状态：当前实现支持 `memberBelow` 堆叠、跟随移动和信任玩家撤回；不会自动把 `memberBelow` 移入休息室
> 最后更新：2026-06-12

## 1. 概述

官方规则中，成员区域支持在成员卡下方重叠能量卡。部分特殊成员卡拥有「可以将成员卡放在自己下方」的效果。本文档描述系统中这类「成员卡下方再夹成员卡」机制，统一称为 `memberBelow`。

本文只维护长期设计边界，不复制字段源码或实现步骤。具体代码以相关路径为准。

## 3. 特殊卡牌识别

部分成员卡可作为 `memberBelow` 的宿主。识别逻辑按卡牌编号模式判断，当前模式由共享工具函数集中维护。

| 卡号模式 | 说明 |
| --- | --- |
| `PL!-bp6-003-*` | 特殊成员卡（bp6 系列） |
| `PL!HS-pb1-002-*` | 特殊成员卡（HS pb1 系列） |
| `PL!N-PR-026-*` | 特殊成员卡（N PR 系列） |

其中 `*` 表示最终编号段。新增特殊成员卡时，应优先更新共享识别工具，而不是在 UI 或处理器中重复写判断。

## 4. 功能边界

### 4.1 堆叠行为

- 来源区域：手牌或休息室。
- 目标区域：已有特殊成员卡的成员槽位。
- 可堆叠卡牌类型：成员卡。
- 堆叠层数：当前不设上限。
- 可用窗口：与己方桌面自由拖拽窗口保持一致。
- 合法性：目标成员必须是特殊成员卡；普通成员槽位不应接受 `memberBelow` 堆叠。

### 4.2 信任玩家撤回

在「信任玩家」原则下，`memberBelow` 中的成员卡可以在自由拖拽窗口内拖回手牌，用于撤回误操作。

这不是标准规则流程中的自动处理，而是当前自由桌面模型下的人工修正能力。正式联机仍应通过命令层和服务端校验记录该移动。

### 4.3 跟随与清理规则

| 场景 | `memberBelow` 行为 |
| --- | --- |
| 特殊成员从槽位 A 移到槽位 B | `memberBelow` 跟随成员一起移动 |
| 特殊成员与另一成员换位 | 双方下方堆叠随各自成员交换 |
| 特殊成员离开成员区 | `memberBelow` 留在槽位，不随主成员自动进入休息室 |
| 新成员占据有 `memberBelow` 的槽位 | 新成员继承该槽位已有的 `memberBelow` |
| `memberBelow` 中的成员卡被拖走 | 从下方堆叠中移除，进入目标区域 |

当前不会自动把 `memberBelow` 清入休息室；需要玩家手动整理。

## 5. 数据模型设计

`memberBelow` 与 `energyBelow` 并行存在于成员槽位数据中，用于记录每个槽位下方堆叠的成员卡实例 ID 列表。

设计原则：

- `memberBelow` 属于成员槽位状态，而不是独立区域。
- `memberBelow` 中的卡牌仍是公开桌面对象。
- 成员槽位移动、换位和离开成员区时，`memberBelow` 的行为应与本文件规则保持一致。
- 联机投影需要把 `memberBelow` 作为成员槽位视图的一部分输出。

## 6. 命令与区域操作

成员卡进入 `memberBelow` 应通过明确的成员堆叠语义表达，而不是普通成员登场或普通跨区移动的隐式副作用。

需要保持的边界：

- 从手牌或休息室堆叠到特殊成员下方时，应带有明确的 `memberBelow` 目标语义。
- 从 `memberBelow` 拖出时，应能从槽位状态中正确移除。
- 成员换位时，`energyBelow` 与 `memberBelow` 都应按各自规则同步处理。
- 服务端应校验来源区域、卡牌类型、目标槽位和自由拖拽窗口。

## 7. 联机投影

联机视图需要把 `memberBelow` 中的卡牌作为公开对象投影：

- 双方玩家都能看到 `memberBelow` 的公开卡牌。
- 对象应保留稳定公开标识，支持连续观察。
- `PlayerViewState` 中的成员槽位视图应包含 `memberBelow` 信息。
- UI 不应通过权威态偷读未投影的信息。

## 8. UI 展示

- 堆叠成员卡以偏移方式显示在特殊成员卡下方。
- 偏移方向与 `energyBelow` 区分，避免视觉混淆。
- 使用不同边框或状态样式区分成员下方堆叠和能量下方堆叠。
- 堆叠数量较多时应显示数量提示，避免卡牌完全遮挡。

## 9. 与 `energyBelow` 的对比

| 维度 | `energyBelow` | `memberBelow` |
| --- | --- | --- |
| 可堆叠卡牌类型 | 能量卡 | 成员卡 |
| 来源区域 | 能量区、能量卡组 | 手牌、休息室 |
| 目标要求 | 槽位有成员卡 | 槽位有特殊成员卡 |
| 跟随移动 | 是 | 是 |
| 离开成员区时的处理 | 依当前成员槽位移动模型处理 | 留在槽位，不自动进入休息室 |
| 可拖回手牌 | 否 | 是，作为信任玩家撤回 |
| 堆叠层数限制 | 无 | 无 |

## 10. 回归关注点

- 手牌成员卡拖到特殊成员下方时，应形成正式堆叠。
- 休息室成员卡拖到特殊成员下方时，应形成正式堆叠。
- `memberBelow` 卡牌拖回手牌时，应从堆叠中移除。
- 特殊成员换位时，`memberBelow` 应跟随。
- 特殊成员离开成员区时，`memberBelow` 应留在槽位。
- 联机视图中双方均可看到公开的 `memberBelow` 卡牌。
