# Loveca 联机模式分阶段重构计划

> 文档类型：实施计划
> 适用范围：把当前本地 `GameSession` / `gameStore` 架构平滑重构到联机首版所需的权威态、事件流与按座位投影模型。
> 最后更新：2026-04-04

---

## Stage 1：联机基础层落地

目标：

- 把联机文档中的核心术语先落成代码，而不是继续停留在草案。
- 保留当前本地 UI 与规则流程，不直接替换现有 `GameState` 渲染链路。

本阶段交付：

- 新增 `Seat / PlayerViewState / PublicEvent` 基础类型。
- 新增权威态到 `PlayerViewState` 的投影器。
- `GameSession` 暴露联机快照出口与基础公共事件时间轴。
- 现有 `getStateForPlayer()` 继续保留，仅作为本地模式兼容接口。

不做的事：

- 不把前端主界面立即迁移到 `PlayerViewState`。
- 不把现有动作协议直接替换成联机命令协议。
- 不实现房间、连接、重连、持久化。

状态：

- 基本完成（兼容态仍在收口）

---

## Stage 2：前端视图迁移与兼容层收口

目标：

- 让前端主渲染逐步从 `GameState` 脱离，改为消费 `PlayerViewState`。

本阶段交付：

- `gameStore` 改为同时持有本地兼容态与 `PlayerViewState`。
- 公共区、隐藏区、盖牌 Live、检视区统一改为按 viewer surface 渲染。
- 现有调试模式与对墙打模式通过同一套投影视图渲染。

风险：

- 组件层改动面大，需要按区域分批迁移。

状态：

- 已完成

---

## Stage 3：命令语义重构

目标：

- 收口规则执行入口，逐步替换“万能拖拽动作”。

本阶段交付：

- 引入联机命令层与命令上下文。
- 按公开移动、检视区操作、声明/确认、过优先权拆分命令。
- `GameSession` 内部统一为 `Command -> Validate -> Apply -> Emit -> Project` 主链路。

状态：

- 进行中（已落地基础命令层、检视上下文与 source-aware 检视命令）

---

## Stage 4：服务端持久化与实时会话

目标：

- 把当前内存会话扩展到可恢复、可同步、可审计的联机实体。

本阶段交付：

- `matches / match_commands / match_events / match_snapshots` 等持久化对象。
- WebSocket 会话层、ack、事件补拉、断线恢复。
- 快照 + 事件回放恢复链路。

状态：

- 待开始

---

## 当前实施原则

- 先新增，不直接推翻现有本地模式接口。
- 先固定投影与事件边界，再改传输层。
- 先让代码结构符合联机模型，再考虑完整自动裁判。
