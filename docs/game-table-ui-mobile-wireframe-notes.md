# 游戏桌移动端低保真页面结构草图说明

> 本文档是对移动端改进方向的进一步细化，用文字线框方式描述页面结构、信息层级和主要交互路径，便于后续进入线框图或视觉设计阶段。

## 1. 说明范围

本说明覆盖以下页面/面板：

- 主桌页
- 卡牌选中后的底部动作条
- 判定任务页
- 资源浏览页
- 分数确认页
- 换牌页
- 更多 / 调试菜单

目标不是定义视觉风格，而是明确：

- 每个页面有哪些区块
- 区块优先级如何
- 用户从哪里进入、如何返回
- 哪些操作应该放在主路径，哪些收纳到次级路径

## 2. 主桌页低保真结构

### 2.1 页面结构总览

推荐竖屏结构：

```text
+--------------------------------------------------+
| 顶部栏：返回 / 回合阶段 / 设置                    |
+--------------------------------------------------+
| 对手摘要栏                                        |
| 名称  手牌数  主卡组  能量  成功Live  回合状态     |
+--------------------------------------------------+
| 对手场上摘要                                      |
| [对手成员槽1] [对手成员槽2] [对手成员槽3]         |
| [对手Live摘要 / 当前公开Live]                     |
+--------------------------------------------------+
| 中央流程提示条                                    |
| 当前阶段 / 子阶段 / 当前建议动作 / 主按钮         |
+--------------------------------------------------+
| 己方Live区                                        |
| [Live槽1] [Live槽2] [Live槽3]                     |
+--------------------------------------------------+
| 己方成员区                                        |
| [左成员槽] [中成员槽] [右成员槽]                  |
+--------------------------------------------------+
| 资源快捷栏                                        |
| [主卡组] [休息室] [能量] [成功区] [判定] [更多]   |
+--------------------------------------------------+
| 手牌区                                            |
| <- 横向滑动卡列 ->                                |
+--------------------------------------------------+
| 底部动作条                                        |
| 根据当前选中对象动态变化                          |
+--------------------------------------------------+
```

### 2.2 顶部栏

顶部栏只保留全局能力：

- 左：返回 / 退出对局
- 中：`T3 Main` 或 `第3回合 主要阶段`
- 右：设置 / 更多

不建议把调试切换、视角切换、主题切换都放在顶部第一层。

### 2.3 对手摘要栏

作用是替代 PC 端完整对手桌面。

建议内容：

- 对手头像或缩写
- 对手名称
- 手牌数
- 主卡组剩余
- 能量数
- 成功 Live 数
- `对手回合 / 你的回合`

交互建议：

- 点击整栏：展开“对手详情抽屉”
- 不在主桌常驻显示对手完整资源区和手牌扇面

### 2.4 对手场上摘要

这一块只保留当前需要公开关注的信息：

- 对手 3 个成员槽
- 对手当前公开 Live

表现建议：

- 成员卡尺寸小于己方
- 对手未公开手牌不在此区域展示
- 如信息过多，优先保留“有卡的槽位”与状态标记

### 2.5 中央流程提示条

这是移动端主流程入口，替代 PC 右下角阶段面板。

建议结构：

```text
+--------------------------------------+
| 表演阶段 · Live判定                   |
| 请先完成应援与 Hearts 判定            |
| [打开判定区]           [撤销]         |
+--------------------------------------+
```

包含：

- 第一行：阶段 + 子阶段
- 第二行：一句当前建议动作
- 第三行：主按钮，必要时显示次按钮

### 2.6 己方 Live 区

Live 区紧贴己方成员区上方，是主战区的一部分。

建议结构：

```text
Live 区
[空槽] [已放置] [成功/失败]
```

状态表达建议：

- 空槽：浅虚线
- 已里侧放置：卡背 + 小角标
- 已公开：缩略正面
- 判定成功：绿色边框
- 判定失败：红色边框

### 2.7 己方成员区

建议维持 3 槽结构，但交互热区做大：

```text
己方成员区
[左槽] [中槽] [右槽]
```

每个槽位内建议能看见：

- 卡图缩略
- 活跃/等待状态角标
- 附加能量数量角标

点击逻辑：

- 点击有卡槽：选中该成员并弹出动作条
- 点击空槽：如果当前选中手牌成员，则直接提示“放入此槽”

### 2.8 资源快捷栏

这一栏用于替代 PC 的左右资源区。

建议结构：

```text
[主卡组 28] [休息室 6] [能量 4/7] [成功 1] [判定] [更多]
```

交互：

- 点击主卡组：进入卡组检视页
- 点击休息室：进入休息室浏览页
- 点击能量：进入能量详情页
- 点击成功：进入成功区浏览页
- 点击判定：进入判定任务页
- 点击更多：打开附加菜单

### 2.9 手牌区

推荐底部横向卡列：

```text
| card | card | card | card | card | ...
```

交互建议：

- 左右滑动浏览
- 单击卡牌：选中并上抬
- 长按卡牌：打开详情页
- 再次点击已选卡：取消选中

不要继续使用 PC 式大扇形。

### 2.10 底部动作条

底部动作条固定存在，但内容动态变化。

未选中对象时：

```text
[查看判定] [结束阶段] [更多]
```

选中手牌成员卡时：

```text
[上场左] [上场中] [上场右] [详情]
```

选中手牌 Live 卡时：

```text
[放入Live] [详情] [取消]
```

选中场上成员时：

```text
[切换状态] [移动] [详情] [收起]
```

选中资源区对象时：

```text
[查看] [抽取/放回] [详情]
```

## 3. 主桌页的典型状态草图

### 3.1 空闲状态

```text
顶部栏
对手摘要
对手场上摘要
流程提示条：你的回合，请完成主要阶段操作
己方Live区
己方成员区
资源快捷栏
手牌区
底部动作条：[判定] [结束阶段] [更多]
```

### 3.2 选中手牌成员卡

```text
手牌中一张卡高亮抬起
成员区 3 个空/可替换槽变亮
底部动作条：[上场左] [上场中] [上场右] [详情]
```

### 3.3 选中场上成员卡

```text
成员卡高亮
底部动作条：[切换状态] [移动] [查看附加能量] [详情]
```

### 3.4 当前处于表演判定阶段

```text
流程提示条变为高亮状态
文案：表演阶段 · 请进行 Live 判定
主按钮：[打开判定区]
底部动作条简化为：[打开判定区] [更多]
```

## 4. 判定任务页低保真结构

移动端判定建议使用全屏页。

### 4.1 页面结构

```text
+--------------------------------------------------+
| 顶部栏：返回 / 标题“Live判定” / 帮助             |
+--------------------------------------------------+
| 当前玩家信息                                      |
| 玩家名 / 主卡组剩余 / 当前Live数量                |
+--------------------------------------------------+
| 应援区                                            |
| [翻开一张] [全部放回/辅助按钮]                    |
| 横向应援卡列                                      |
+--------------------------------------------------+
| Hearts汇总                                        |
| 各颜色 Hearts、加成、说明                         |
+--------------------------------------------------+
| Live需求区                                        |
| Live1 需要xx                                      |
| Live2 需要xx                                      |
+--------------------------------------------------+
| 底部固定操作区                                    |
| [LIVE失败] [LIVE成功] 或 [成功效果发动完毕]       |
+--------------------------------------------------+
```

### 4.2 应援区交互

建议每张应援牌点击后弹出小动作表，而不是 hover 菜单：

- 加入手牌
- 放入休息室
- 放回卡组顶
- 查看详情

### 4.3 返回逻辑

返回行为分两种：

- 如果只是辅助查看，可直接返回主桌
- 如果当前子阶段要求必须完成判定，则返回前给出提示

## 5. 资源浏览页低保真结构

资源浏览建议统一为可复用模板。

### 5.1 休息室页

```text
顶部栏：返回 / 休息室 / 数量
筛选或排序栏（可选）
网格卡牌列表
底部动作条：根据选中牌变化
```

选中卡牌后的底部动作条示例：

- `加入手牌`
- `放回卡组顶`
- `详情`

### 5.2 成功区页

```text
顶部栏：返回 / 成功Live区 / 数量
纵向或横向卡牌列表
卡牌点击后查看详情
```

### 5.3 能量页

```text
顶部栏：返回 / 能量区 / 活跃数-总数
能量卡列表
每张卡显示活跃/等待状态
```

### 5.4 主卡组检视页

建议延续“翻开卡组顶”逻辑，但改成移动端更直接的布局：

```text
顶部栏：返回 / 检视主卡组
按钮行：[翻开一张] [放回顶部] [全部放回]
检视卡列（横向）
底部动作条：对选中检视牌操作
```

不建议在移动端继续强依赖拖拽排序，可改为：

- 左移
- 右移
- 放回顶部
- 放到底部

## 6. 分数确认页低保真结构

### 6.1 页面结构

```text
+--------------------------------------------------+
| 顶部栏：结果结算 / 说明                           |
+--------------------------------------------------+
| 双方分数卡片                                      |
| 己方分数                                          |
| 对手分数                                          |
+--------------------------------------------------+
| 己方分数调整输入                                  |
+--------------------------------------------------+
| 当前确认状态                                      |
| 己方：已确认/未确认                               |
| 对手：已确认/未确认                               |
+--------------------------------------------------+
| 底部主按钮：[确认我的分数]                        |
+--------------------------------------------------+
```

这页应尽量简单，不混入其它复杂操作。

## 7. 换牌页低保真结构

### 7.1 页面结构

```text
顶部栏：换牌阶段 / 先攻或后攻
说明区：可选择0-6张牌换掉
手牌网格或横向大卡列
已选择数量提示
底部按钮：[不换牌] [确认换牌]
```

### 7.2 交互建议

- 单击卡牌：选中 / 取消选中
- 选中后明显描边
- 顶部持续显示“已选择 N 张”

## 8. 更多 / 调试菜单低保真结构

当前很多能力不应直接常驻主桌，可统一放入更多菜单。

### 8.1 菜单内容建议

正式模式下：

- 查看日志
- 切换主题
- 查看对手详情
- 帮助

调试模式下追加：

- 切换视角
- 切换对墙打 / 调试模式
- 自由移动卡牌
- 快速抽牌 / 放回

### 8.2 展现形式

建议用底部 action sheet：

```text
更多
- 查看日志
- 查看对手详情
- 调试选项
- 取消
```

## 9. 页面之间的跳转关系

推荐主路径：

```text
主桌页
  -> 判定页
  -> 主卡组检视页
  -> 休息室页
  -> 成功区页
  -> 能量页
  -> 分数确认页
  -> 换牌页
  -> 更多菜单
```

设计原则：

- 所有重任务都从主桌进入
- 完成后回到主桌
- 不形成过深层级

## 10. 交互优先级建议

### 一级操作

这些应该在主桌一跳完成：

- 选中手牌
- 上场 / 放入 Live
- 切换成员状态
- 结束阶段
- 打开判定

### 二级操作

这些可以进入抽屉或二级页面：

- 查看详情
- 浏览休息室
- 浏览成功区
- 检视卡组
- 能量查看

### 三级操作

这些应收纳到更多菜单或调试面板：

- 视角切换
- 模式切换
- 日志
- 自由移动
- 高级检视排序

## 12. 结论

移动端低保真结构的关键是：

- 主桌简化
- 战场聚焦
- 资源收纳
- 底部动作条承担主交互
- 判定与结算进入全屏任务页

如果你后续要继续推进，我建议下一步直接出一版“移动端主桌 + 判定页”的 ASCII 线框图，或者继续整理成组件级信息架构说明。
